Step 2: Management Interface¶

When building our UI logic, we need to consider a special case for our management interface. We do not want the users to be able to disable this interface.

To accomplish this, we will add logic again using the Django template language to exclude that interface from displaying an Enable/Disable button.

To simplify the logic inside our HTML code, we will add another if statement in our Django template to include a different HTML file for the interface depending on the interface name.

Let’s first extract our existing logic for the <tr> (table’s interface row) to a separate file named interface.html in the interfaces/templates/interfaces/ directory.

We will now create a simplified version (without the Enable/Disable button) for our management interface, we will name this file management_interface.html and it will have the following contents:

We will now replace the <tr> inside table.html with a Django template if statement to include the file depending our interface name:

interfaces/templates/interfaces/table.html¶
{% for interface in interface_list %}
<tr>
    {% if interface.name == "GigabitEthernet1" %}
    {% include "./management_interface.html" %}
    {% else %}
    {% include "./interface.html" %}
    {% endif %}
</tr>
{% endfor %}
